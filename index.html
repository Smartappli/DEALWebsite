One suite, four components

Build, host, govern and operate.

Each component can be adopted independently. DEALInterface is the fourth component: the unified console that coordinates field ingestion, operated infrastructure, governance, controlled data sharing and operator workflows.

01

DEALIoT

Real-time IoT platform for sensors, media metadata and industrial streams. MQTT, Kafka, Flink, Airflow and Kubernetes are assembled to move from prototype to scalable operations.

  • MQTT to Kafka ingestion
  • Stream processing and backfill
  • Integrated observability and DLQ
View DEALIoT on GitHub
02

DEALHost

Operated hosting for critical data workloads: Kubernetes, object storage, monitoring, backups, hardening and continuous operations.

  • Secured application clusters
  • Runbooks, alerts and backups
  • Cloud, hybrid or on-prem paths
View DEALHost on GitHub
03

DEALData

Governance and value layer: catalogues, evidence topics, controlled exports, Data Act, DGA, FAIR, Zenodo and OpenAIRE.

  • Catalogues and data products
  • Controlled sharing and traceability
  • Usable compliance evidence
View DEALData on GitHub
04

DEALInterface

Unified management console for the DEAL suite. Product modules stay isolated while IAM, RBAC, audit, billing, support and operator workflows are centralized.

  • Single control plane
  • IAM, RBAC and audit surfaces
  • Operator workflows and live probes
View DEALInterface on GitHub

Direct answers

Short answers for fast evaluation.

This section gives decision-makers, integrators and answer engines the canonical definitions of DEALIoT, DEALHost, DEALData and DEALInterface.

What is DEALIoT?

DEALIoT is a production-oriented IoT data platform for MQTT ingestion, Kafka transport, schema governance, stream processing, backfill and operational evidence.

What is DEALHost?

DEALHost operates the DEAL suite on hardened infrastructure: Kubernetes, monitoring, backups, runbooks, security and support boundaries.

What is DEALData?

DEALData structures catalogues, data products, controlled sharing, FAIR exports, Zenodo, OpenAIRE and Data Act or Data Governance Act evidence.

What is DEALInterface?

DEALInterface is the unified management console for the DEAL suite: it centralizes IAM, RBAC, audit, billing, support and operator workflows while preserving module boundaries.

How do I start an evaluation?

The recommended path is to run the local smoke test, choose one representative field stream, scope a 30-day pilot and score the evidence before production.

Which use cases fit?

The suite targets research teams, precision agriculture, connected livestock, industrial IoT, public projects and integrators that must industrialize field data.

Is the platform compliance-certified?

The repository provides compliance evidence, workflows, topics and templates. These assets support audits, but do not replace organization-specific certification.
